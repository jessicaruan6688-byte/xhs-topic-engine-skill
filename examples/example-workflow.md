# 完整工作流示例

## 场景：下 7 天发什么？

### 用户输入

```
帮我找下一周职场赛道的选题
```

### Skill 执行过程

**Step 1**：读取 account-profile.md
- 人设：求职/转岗型 + 女性成长型
- 种子词：裸辞、背调、谈薪、空窗期、30岁转行

**Step 2**：搜索评论区信号
- 扫了 5 篇爆款笔记评论区
- 提取高频词：背调(23)、空窗期(18)、裸辞(15)、谈薪(12)
- 发现争议点：「要不要告诉面试官我在空窗期」

**Step 3**：搜索联想词
- 「空窗期」→ 空窗期怎么说、空窗期多久正常、空窗期被刷
- 「30岁」→ 30岁转行、30岁没存款、30岁从头来过

**Step 4**：评分卡打分
- 「空窗期怎么回答」评分：44/50 → 马上做
- 「裸辞后悔了」评分：38/50 → 马上做
- 「30岁转行晚不晚」评分：35/50 → 马上做

**Step 5**：输出 TOP5 选题

```
📋 下周选题 TOP5

1️⃣ 【空窗期怎么回答，HR才不会刷掉你】
   类型：工具清单型 | 评分：44 | 私域钩子：完整话术→私域
   周一晚 8 点发

2️⃣ 【裸辞 3 个月了，我后悔了吗？】
   类型：真实经历型 | 评分：38 | 私域钩子：无（先建信任）
   周三晚 8 点发

3️⃣ 【30 岁转行，A 去读书 B 去创业 C 继续干】
   类型：选择题互动型 | 评分：35 | 私域钩子：免费测评→私域
   周五晚 8 点发

4️⃣ ...
5️⃣ ...
```

## 场景：研究一个博主

### 用户输入

```
研究一下 @职场老张
```

### Skill 执行过程

1. 搜索该博主近 7 天笔记
2. 逐篇拆解（标题公式、封面、正文结构、互动方式、私域钩子）
3. 输出报告：

```json
{
  "博主": "@职场老张",
  "人设": "HR/管理者型",
  "近7天笔记数": 5,
  "平均互动": 3400,
  "爆款率": "40%",
  "核心公式": {
    "标题": "痛点 + 权威背书（"HR告诉你，面试别做这3件事"）",
    "封面": "真人出镜 + 大字报标题",
    "正文": "开头一句话立场 → 3-5条清单 → 评论区互动"
  },
  "可复用的": ["标题公式", "互动话术"],
  "他的短板": "私域引流方式太单一，只在简介留微信"]
}
```
