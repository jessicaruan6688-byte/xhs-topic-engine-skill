# 活人感写作原则

> **为什么大部分 AI 写的职场内容像屎？因为太完美了。**

---

## 原则一：必须露怯

```
❌ AI写法：「掌握这5个技巧，你就能拿到所有offer」
✅ 活人写法：「我面了12家挂了11家，第12家是我用这个方法拿下的」

❌ AI写法：「高效职场人士的7个习惯」
✅ 活人写法：「上班第三天就想辞职了，我是不是不适合打工」
```

> **原则**：先说自己的失败，再说自己的方法。失败越具体，用户越信。

### 露怯的 3 种方式

| 方式 | 例子 |
|------|------|
| 承认能力不足 | "我真的不会谈薪，每次都说'您看给多少就多少'" |
| 承认情绪失控 | "那天我在厕所哭了15分钟，然后擦干脸继续面试" |
| 承认判断失误 | "我后悔了，当初不该为了情怀降薪去那家公司" |

---

## 原则二：必须有不完美细节

```
❌ 「我和HR谈薪，用了这个方法，薪资涨了30%」

✅ 「那天我手心全是汗，HR问期望薪资的时候，我说了句蠢话：
    '您看……公司一般给多少？'
   说完我就后悔了。但没想到……」
```

> **原则**：加一个让你看起来不专业的细节（手心出汗、说错话、犹豫了）。
>
> 完美是 AI，不完美是人。

### 不完美细节库（直接用）

```
「手抖得打不开电脑」
「当时脑子一抽说了句……」
「我也不知道哪来的勇气，直接说了不」
「面试官笑了一下，我心想完了」
「低头一看，衬衫扣子扣错位了」
「说出来不怕你们笑，我当时连JD都没看懂」
```

---

## 原则三：必须留一个「没说完」的口子

```
❌ 「这就是全部方法了，收藏吧」

✅ 「剩下的我还没试过，等我面完下一家来汇报」
✅ 「这个话术对不同公司可能要微调，你们面的是什么类型的公司？」
```

> **原则**：不要给完美答案。留一个缺口 = 留一个评论区互动的入口。

### 常用「留口子」句式

| 场景 | 句式 |
|------|------|
| 没说完 | "第三点我还在试，有用的话回来更新" |
| 求补充 | "你们公司的情况可能不一样，评论区说说？" |
| 留悬念 | "下周我面另一家，到时候来汇报结果" |
| 抛问题 | "同样的处境，你会怎么选？" |

---

## 原则四：图片中的字体得是人的字体

小红书封面最明显的 **「AI 味」**：

| AI 味特征 | 活人感替换 |
|-----------|-----------|
| 字体太规整（思源黑体、苹方） | 手机备忘录截图 / 手写体 |
| 排版太对称 | 留白不均、文字歪一点 |
| 颜色太干净 | 用醒图加滤镜、胶片感 |
| 配图太精致（插画/3D） | 实拍、截图、粗糙的拼图 |
| 每张图都有统一模板 | 每期换一种排版风格 |

### 快速出活人感封面的工具流

```
手机备忘录截图 → 醒图加背景 → 加粗体文案 → 发
```

不要让 AI 生成封面。封面必须用手机/电脑原生工具做。

## 原则五：用口语写作

### 降维对照表

| AI 感词汇 | 活人感替换 |
|-----------|-----------|
| 首先、其次、最后 | 第一、第二、第三 / 先说、再说、最后说 |
| 综上所述 | 所以 / 总的来说 |
| 值得注意的是 | 划重点 / 注意了 |
| 至关重要 | 特别重要 / 一定要看 |
| 解决方案如下 | 我是这么搞的 |
| 通常情况下 | 大部分时候 |
| 例举几个案例 | 举个栗子 |

### 口语化的 3 个检查清单

1. **[ ]** 你会在饭桌上跟朋友这么说吗？
2. **[ ]** 去掉所有带「的」的长修饰词了吗？
3. **[ ]** 有没有一句话超过 30 个字？拆成两句。

## 六、私域钩子原则

### 不能太硬

```
❌ 「加我微信领资料」
✅ 「完整版我放在[xxx]，你在评论区打个'求'也可以，我私你」
```

### 先给价值，再给入口

```
正确的顺序：
1. 这篇给了你 20% 的价值
2. 评论区互动（你告诉我你的情况）
3. 私域获取剩下 80% 的价值
```

### 私域钩子类型一览

| 类型 | 文案示例 | 转化率 |
|------|---------|:------:|
| 资料包 | "完整50页谈薪话术→私域入口" | ⭐⭐⭐⭐⭐ |
| 免费测评 | "测测你的性格适合什么工作→私域入口" | ⭐⭐⭐⭐ |
| 1v1咨询 | "具体情况可以私信聊" | ⭐⭐⭐ |
| 社群入口 | "我们有个群专门聊职场转型" | ⭐⭐⭐⭐ |
| 直播预告 | "本周六直播聊面试技巧→私域入口" | ⭐⭐⭐ |
